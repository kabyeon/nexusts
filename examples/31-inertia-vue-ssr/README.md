# 31 · Inertia + Vue 3 (SSR)

Inertia.js v3 with **Vue 3** in **server-side rendered** mode. The
"Laravel + Inertia (SSR) with Vue" recipe.

## What it shows

- `createVueAdapter` SSR adapter plugged into `new Inertia({ ssr })`
- Server renders the Vue component tree to HTML via `@vue/server-renderer`
- Client uses `createApp` which auto-hydrates when target DOM has rendered HTML
- Same page component (`HomePage`) is shared between server and client

## How to run

```bash
cd examples/31-inertia-vue-ssr
bun install                                          # vue + @inertiajs/vue3 + @vue/server-renderer
bun build ./frontend/client.js --outdir=./public \
  --target=browser --format=esm --minify
bun main.ts
```

Open <http://localhost:3000> and view-source. The response body
contains the rendered `<h1>Hello from Inertia + Vue SSR!</h1>`,
not a `data-page` JSON string.

## Dev mode

```bash
bun build ./frontend/client.js --outdir=./public \
  --target=browser --format=esm --watch &
bun main.ts
```

## Code

### Server — `main.ts`

```ts
import { Inertia, createVueAdapter } from "@kabyeon/nexusjs/view";
import { HomePage } from "./frontend/home.js";

@Module({
  providers: [{
    provide: Inertia.TOKEN,
    useValue: new Inertia({
      ssr: createVueAdapter({ components: { Home: HomePage } }),
    }),
  }],
  controllers: [HomeController],
})
class AppModule {}
```

### Shared page component — `frontend/home.js`

```js
export const HomePage = {
  props: ["greeting", "count", "errors"],
  template: `
    <main>
      <h1>{{ greeting }}</h1>
      <p>Counter: {{ count }}</p>
      <a href="/counter" method="post" as="button">+1</a>
      <form action="/greet" method="post">
        <input name="name" />
        <p v-if="errors?.name" style="color: crimson;">{{ errors.name }}</p>
        <button>Submit</button>
      </form>
    </main>
  `,
};
```

### Hydration entry — `frontend/client.js`

```js
import { createApp, h } from "vue";
import { createInertiaApp } from "@inertiajs/vue3";
import { HomePage } from "./home.js";

createInertiaApp({
  resolve: (name) => HomePage,
  setup({ el, App, props, plugin }) {
    createApp({ render: () => h(App, props) })
      .use(plugin)
      .mount(el);
  },
});
```

## How to verify SSR is working

```bash
curl -s http://localhost:3000/ | grep -A 1 "<div id=\"app\""
```

If the response contains your component's HTML **inside** `<div id="app">`,
SSR is working. If you only see `data-page="..."` with an empty `<div>`,
the SSR adapter failed and the framework fell back to the default shell.
